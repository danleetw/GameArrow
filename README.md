# 弓箭手對決 Arraw Duel

第一人稱弓箭手對戰——玩家 vs AI 的單人對決遊戲。用 [three.js](https://threejs.org/) 純前端渲染，沒有後端伺服器，沒有外部 3D 模型／貼圖，人物、地形、生物、天空全部程式化生成。

**線上遊玩**：https://danleetw.github.io/GameArrow/

## 操作方式

| 操作 | 功能 |
|---|---|
| 滑鼠移動 | 轉動視角瞄準（需先點一下畫面鎖定滑鼠） |
| 左鍵按住 / 放開 | 蓄力拉弓 / 放開射箭（蓄力越久箭速越快） |
| 右鍵按住 + 滾輪 | 望遠縮放視野（純視覺放大，放開右鍵立即恢復，不影響實際瞄準方向） |
| `Esc` / 空白鍵 | 暫停畫面中恢復遊戲 |
| `M` | 開關背景音樂 |
| `B` | 除錯用：在附近觀戰高台上排開 5 隻鳥懸停，方便近距離看鳥的外觀/動畫 |
| `G` | 除錯用：在空地上生成一隻殭屍（會自動避開水域/樹木/石頭） |

暫停選單裡另外有「開場運鏡開關」設定（存在瀏覽器 `localStorage`，記得上次選擇），以及重新來過、回到選單等按鈕。

## 玩法簡介

雙方站定原地面對面，命中對手身體不同部位造成不同傷害（頭/胸致命、大臂/大腿重傷、小臂/小腿中傷、手掌/腳掌輕傷），並依嚴重程度打亂瞄準準度、甚至中斷正在蓄力的動作，直到一方陣亡為止。場景中另外有動態風力、會停棲的鳥類生態、會主動攻擊雙方的殭屍，以及跟真實日期連動的日夜循環。

## 技術棧

| 分類 | 選用 |
|---|---|
| 3D 渲染 | [three.js](https://threejs.org/) — 場景、相機、幾何體、材質、光影、陰影 |
| 建置工具 | [Vite](https://vitejs.dev/)（`base: './'`，打包後可用相對路徑部署到任何子目錄，例如 GitHub Pages） |
| 語言 | JavaScript ES Modules（無 TypeScript） |
| 物理 | 箭矢/鳥/殭屍皆為**手動運動學積分**（位置＋速度＋重力自行逐幀計算），刻意不用剛體物理引擎，避免細長高速物體的穿透（tunneling）問題；`cannon-es` 僅列於相依套件，核心玩法未實際使用 |
| 音效 | Web Audio API 原生振盪器/雜訊緩衝即時合成拉弓/放箭/命中/死亡等短音效；背景音樂另外載入 `sound/` 目錄下的外部 mp3（[Pixabay](https://pixabay.com/) 授權） |
| 美術資源 | 無外部 3D 模型／貼圖，人物、弓箭、鳥、殭屍、地形、樹木、水面、天空全部用 `THREE.*Geometry` 組合＋`CanvasTexture` 手繪貼圖程序化生成 |
| 持久化 | `localStorage`，僅存開場運鏡開關等本機設定，無後端、不收集任何使用者資料 |
| 部署 | GitHub Actions 在每次推送到 `main` 時自動 `npm run build` 並發布到 GitHub Pages |

## 物件邏輯

### 🐦 鳥類生態（`src/birds.js`）

氛圍用的環境鳥，純裝飾、不影響玩法，同時最多 5 隻。狀態機：`FLY`（飛行）／`CIRCLE`（盤旋）／`APPROACH`（飛去降落點）／`PERCHED`（停棲）／`APPROACH_ARCHER` / `PERCH_ARCHER`（飛去/停在雙方頭上或身邊地面）／`DEAD`。

- 每隻鳥飛一段時間後會重新擲骰決定下一個行為：繼續飛（可能轉向）、原地盤旋、飛去樹梢、飛去空地、飛去觀戰高台，或飛到某個角色頭上/身邊落地。
- **降落動畫**：接近降落點時翅膀收起、雙腳伸出，並把身體墊高讓腳掌貼齊停棲面（地面/樹梢/平台/角色）；落地那一刻姿勢會轉正成水平站姿，不會維持俯衝時的頭低尾高角度。
- 被箭矢射中會直接死亡定格插在箭上（還在飛的話會先落到地面再靜止），靠近的箭矢也會嚇飛正停棲在角色身上的鳥。

### 🧟 殭屍（`src/zombie.js`）

按 `G` 生成，會避開水域/障礙物。每隻殭屍是獨立的簡易狀態機：`mode ∈ {wander, seek, attack, leave}` 決定移動意圖，`state ∈ {active, dying, corpse}` 決定生死階段。

- **一般攻擊**（`seek`，自己巡邏靠近咬到的，或警戒範圍內被動發現）：咬 1~2 口就離開。
- **反擊/復仇**（`attack`，被箭射中觸發）：鎖定射他的那個人，持續攻擊直到把對方咬死才離開；中箭滿 3 箭死亡。
- **警戒規則**：10 公尺內有人正在拉弓蓄力必定被盯上；沒人拉弓時，10 公尺內每秒仍有 1/8 機率、5 公尺內提高到 1/4 機率被動盯上附近的人。
- **免疫保護**：任何一方累積被咬滿 2 口後，接下來 60 秒內所有殭屍都不會再攻擊或鎖定他，逼近中的殭屍會直接放棄轉為離開，避免被無限連咬。
- 死亡倒地後 3 秒自動消失；場上同時存在上限 10 隻（只算活著的）。

### 🎯 AI 對手（`src/ai.js`）

跟玩家共用同一個 `Archer` 類別，狀態機：`IDLE`（待機）→`DRAWING`（鎖定目標蓄力）→`COOLDOWN`（出手後停頓）。

- 用封閉解的拋物線公式反推發射仰角（依目標距離/高度差/箭速），瞄準點會依機率偏向頭/胸要害。
- **命中機率固定 1/10**：每次拉弓先擲骰決定這一箭是不是「真的瞄準」的那 1/10（用極小誤差，幾乎必中），其餘 9/10 直接加一個保證偏出目標範圍的角度，而不是疊加連續隨機誤差（那樣命中率會隨距離飄動，沒辦法保證固定比例）。
- 拉弓中被射中會嚇一跳，瞄準方向被打亂上偏（依傷害等級決定偏多少）。
- 偵測到 11 公尺內有活著的殭屍時會延後蓄力，避免被殭屍盯上。

### ☀️🌙 太陽、月亮與日夜循環（`src/daynight.js`）

現實時間 128 倍速換算成遊戲時間：現實每 11.25 分鐘＝遊戲一整天（24 小時）。

- 太陽與月亮永遠在天空弧線的兩端（互為正對面），只有仰角在地平線以上的那個會發光、投影；仰角越低強度越低、越接近日出日落的暖色調。
- **月相是真實的**：用已知新月時刻＋朔望月週期（29.530588853 天）算出「今天」對應的真實月相（新月/眉月/上弦/凸月/滿月……），並依北半球（台灣）視角正確畫出彎月形狀（漸盈時右側亮、漸虧時左側亮）。
- 負責投影的主光源永遠是仰角較高的那個天體——白天用太陽、夜晚用月亮，換手發生在兩者仰角都貼近地平線的瞬間，那一刻強度也接近 0，陰影方向切換不會有明顯跳動。
- 月光強度跟著月相走（滿月最亮、新月最暗）但保留下限；夜晚的環境光/半球光也設有亮度下限，避免畫面全黑看不清楚場景。
- 太陽/月亮貼圖錨定在攝影機目前位置（而不是場地原點），代表無限遠的天體，不會因為玩家站位偏移而產生視差。

### 💨 風（`src/wind.js`）

風向角與風速用帶阻尼的隨機漫步（damped random walk）緩慢漂移，不會忽快忽慢地跳動。影響箭矢飛行的側向加速度、樹木/草叢的搖晃幅度、風向旗的指向。

## 開發與執行方式

```bash
npm install       # 安裝相依套件
npm run dev        # 啟動 Vite dev server
npm run build       # 打包輸出到 dist/
npm run preview      # 預覽打包後的結果
```

全程不需後端，純前端本機即可完整遊玩測試。
